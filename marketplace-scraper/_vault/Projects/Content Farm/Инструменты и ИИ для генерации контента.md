# Инструменты и ИИ для генерации контента

## Принцип

Не нужен один "супер-ИИ".

Нужна цепочка инструментов:

```text
сбор -> нормализация -> скоринг -> анализ -> драфт -> адаптация RU/UA -> визуал -> approval -> публикация -> метрики
```

## Базовый технический стек

### Backend / сбор

- Python
- SQLite для MVP
- `requests` или `httpx`
- `feedparser`
- `beautifulsoup4`
- `PyYAML`
- `python-dateutil`
- `loguru`
- `python-dotenv`
- `pytest`
- `ruff`

Уже используется:

- `D:\Development\Parsers\RedditScraper`
- `google-generativeai`

### Позже

- PostgreSQL, если SQLite станет тесным.
- Qdrant или Chroma, если понадобится векторный поиск.
- n8n для расписания.
- Docker для стабильного запуска.

## Источники / API

### MVP

- Reddit через аккуратный low-rate collection или официальный путь, где доступен.
- RSS feeds.
- Hacker News official Firebase API.
- Product Hunt API только с учётом ограничений коммерческого использования.
- GitHub API.

### Не MVP

- X API: только официальный API, без browser automation.
- Meta/Instagram/Facebook: не использовать как source scraping. Использовать как publishing channel.

## LLM для анализа и текстов

Нужны роли, а не просто модели:

### 1. Signal Analyst

Задача:

- выбрать важные источники;
- объяснить почему они важны;
- найти pain/buyer intent;
- поставить risk flags.

Подойдут:

- Gemini;
- Claude;
- GPT-class models.

### 2. Editor / Copywriter

Задача:

- Telegram post;
- X post;
- LinkedIn post;
- Instagram/Facebook caption;
- RU/UA adaptation.

Важно:

- не копировать источник;
- не давать нерелевантный hype;
- сохранять ссылку на источник внутри системы.

### 3. Translator / Localizer

Задача:

- адаптация между русским и украинским;
- не дословный перевод;
- локальные примеры;
- правильный тон.

### 4. Risk Checker

Задача:

- найти medical/legal/finance/gambling/crypto-trading risk;
- отметить unsupported claims;
- запретить публикацию high-risk drafts без ручной проверки.

### 5. Visual Prompt Writer

Задача:

- prompt для изображения;
- prompt для short video;
- обложка/thumbnail idea;
- визуальная метафора без текста на картинке, если это важно для платформы.

## Embeddings / память

Для MVP можно не начинать с embeddings.

Добавить позже, когда накопится 500+ source items/drafts.

Использование:

- поиск похожих тем;
- дедупликация;
- "мы уже писали об этом?";
- кластеризация трендов.

Варианты:

- OpenAI embeddings;
- Gemini embeddings;
- локальные embeddings через `sentence-transformers`;
- BGE / E5 / Nomic embedding models.

## Генерация изображений

Использовать только после появления стабильных форматов.

Варианты по задачам:

- простые карточки: Canva / Figma / HTML templates;
- AI-иллюстрации: Midjourney / Ideogram / Flux / Imagen-подобные модели;
- обложки: отдельный шаблон + генерация фона;
- графики: Python/HTML screenshots.

Для AI/SaaS лучше не злоупотреблять "futuristic robot brain neon".

Лучше:

- чистые схемы;
- интерфейсные mockups;
- метафоры автоматизации;
- карусели с тезисами;
- before/after workflow.

## Генерация видео

MVP:

- текстовый сценарий;
- CapCut/ручная сборка;
- простые шаблоны Reels/Shorts.

Позже:

- автоматический storyboard;
- TTS;
- subtitles;
- шаблонный монтаж;
- batch render.

Подходящие типы:

- 30-60 секунд;
- один тезис;
- 3-5 экранов;
- один CTA.

## TTS / Voice

Для RU/UA контента:

- тестировать качество украинского отдельно;
- проверять ударения;
- не публиковать без прослушивания.

Можно использовать:

- Gemini/OpenAI/ElevenLabs/локальные TTS, если качество подходит;
- уже существующий опыт из `SileroTTS`.

## Публикация / scheduling

### Telegram

Самый простой канал для автоматизации:

- Telegram Bot API;
- n8n;
- ручное approval.

### X

Только официальный API или ручная публикация.

Не использовать browser automation.

### Instagram/Facebook

Начать с:

- Meta Business Suite;
- ручное/полуручное планирование.

Потом:

- официальный API;
- проверенный scheduler.

### LinkedIn

Лучше сначала ручной постинг.

LinkedIn плохо переносит спамный автоматический контент.

## Минимальный набор для старта

1. Python + SQLite.
2. Обновлённый `RedditScraper`.
3. RSS + HN + Product Hunt/GitHub fetchers.
4. Gemini API для MVP-драфтов.
5. Obsidian export queue.
6. Telegram как первый канал публикации.
7. Meta Business Suite для Instagram/Facebook без API на старте.
8. Таблица метрик.

## Не покупать/не строить сразу

- прокси-ферму;
- браузерных ботов;
- много аккаунтов;
- автокомментинг;
- автолайки;
- полноценную веб-панель;
- дорогой video generation pipeline.

Сначала доказать, что source -> draft -> post даёт реакцию.
