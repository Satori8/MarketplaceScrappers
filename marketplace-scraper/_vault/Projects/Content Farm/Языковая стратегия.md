# Языковая стратегия

## Планируемые языки

- русский;
- украинский.

## Важное решение

Лучше не мешать языки хаотично в одном аккаунте, если цель - быстрый рост алгоритмов и понятное позиционирование.

Варианты:

## Вариант A: два отдельных бренда

Один бренд на русском, второй на украинском.

Плюсы:

- понятная аудитория;
- меньше путаницы в алгоритмах;
- проще держать тон;
- легче тестировать офферы.

Минусы:

- в 2 раза больше операционной нагрузки;
- нужны отдельные контент-планы.

## Вариант B: один бренд, два языка

Один аккаунт, часть постов на русском, часть на украинском.

Плюсы:

- проще стартовать;
- вся аудитория в одном месте.

Минусы:

- слабее позиционирование;
- часть аудитории будет пропускать часть контента;
- алгоритмам сложнее понять, кому показывать.

## Вариант C: один основной язык + адаптации

Стартовать на русском или украинском, а второй язык использовать для лучших постов.

Плюсы:

- хороший компромисс;
- меньше нагрузки;
- можно быстро понять, где лучше метрики.

Рекомендация для старта:

> Начать с одного основного языка на 30 дней, а лучшие 20% постов адаптировать на второй язык.

## Как адаптировать

Не переводить дословно.

Адаптация должна менять:

- примеры;
- культурный контекст;
- CTA;
- термины;
- уровень формальности.

## Украинский контент

Сильные направления:

- малый бизнес;
- e-commerce;
- автоматизация;
- AI tools для предпринимателей;
- волонтёрские/прикладные технологии;
- локальная адаптация западных трендов.

## Русский контент

Сильные направления:

- AI tools;
- SaaS;
- no-code/automation;
- digital-монетизация;
- productivity;
- тренды из Reddit/X.

## Практическое правило

Каждый content item хранить в нейтральной структуре:

```text
source idea -> core insight -> RU draft -> UA draft -> platform variants
```

Так один сигнал можно переиспользовать без копипаста.
