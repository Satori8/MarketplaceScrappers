=== СТРУКТУРА СКРИПТА (8 блоков) ===

БЛОК 1 — HOOK
- Провокационное наблюдение или вопрос
- Самоидентификация зрителя ("это про тебя")
- Без вступлений, сразу в суть

БЛОК 2 — PROMISE
- Что узнает зритель
- Почему это важно
- Лёгкая лесть ("если ты это смотришь — ты не такой как все")

БЛОК 3-4 — SCIENTIFIC EXPLANATION (повторяется 2-4 раза)
- Психологический термин → простое объяснение
- Ссылка на исследование или учёных (без точных цитат)
- Конкретная дата или сцена для якоря

БЛОК 5 — STORY / IMAGE
- Конкретная бытовая сцена или метафора
- Зритель должен узнать себя или кого-то близкого

БЛОК 6 — PARADOX / TWIST
- Неожиданный угол ("всё не так как кажется")
- Переворот ожиданий

БЛОК 7 — PRACTICE / APPLICATION
- Конкретные шаги или наблюдения
- Нумерованные списки или "habit stacking"

БЛОК 8 — CLOSING + CTA
- Философский или эмоциональный вывод
- Призыв подписаться + переход на следующее видео

=== СТИЛЬ И ТОН ===
- Язык: английский, разговорный, без академизма
- Юмор: 1 момент на каждые 2-3 минуты
  * Самоирония / узнавание себя
  * Абсурдное преувеличение
  * Неожиданный бытовой образ
  * Лёгкая ирония над "экспертами"
- Риторические вопросы каждые 2-3 абзаца
- Прямое обращение к зрителю ("you", "your brain")
- Знаки ! ? ... для оживления монотонного TTS
- НЕ использовать спецсимволы: * [ ] < > и подобные

=== ФОРМАТ ВЫВОДА — ТРИ ФАЙЛА ===

На каждый скрипт генерируется ОБЯЗАТЕЛЬНО ТРИ файла:

━━━━
ФАЙЛ 1: [slug].html  — РЕЖИССЁРСКАЯ ВЕРСИЯ (для человека)
━━━━
Полная читаемая версия скрипта. Никаких технических параметров.
Содержит:
- Title, Description, Tags, Thumbnail prompt
- ПЕРВАЯ СЦЕНА = thumbnail сцена (см. правила ниже)
- Полный скрипт по блокам
- Описание сцен в свободной форме (что происходит, какой персонаж, какое настроение)
- Текст озвучки построчно
- Паузы отмечены как [pause]
- Тип сцены отмечен читаемо: [СТАТИКА], [АНИМАЦИЯ], [ЗАЦИКЛЕННАЯ АНИМАЦИЯ]
- Описание движения камеры в свободной форме: "камера медленно приближается к лицу персонажа"
- Описание анимации в свободной форме: "персонаж медленно поднимает руку и машет"
НЕ содержит: JSON, технических параметров, числовых значений scale/easing/fps

━━━━
ФАЙЛ 2: [slug]_strings.txt  — ТЕКСТ ДЛЯ TTS (для Python-скрипта озвучки)
━━━━
Правила:
- ТОЛЬКО текст озвучки, ничего лишнего
- БЕЗ промптов, БЕЗ пауз, БЕЗ HTML, БЕЗ заголовков
- Каждая строка = один аудиофайл
- Строки разбиваются только на длинных паузах (смена блока, переход между пунктами)
- Короткие паузы внутри строки — через "." или "..."
- Пустых строк нет

━━━━
ФАЙЛ 3: [slug]_scenes.json  — ТЕХНИЧЕСКИЕ ДАННЫЕ ДЛЯ ПАЙПЛАЙНА
━━━━
Машиночитаемый файл. Парсится Python-скриптом для генерации изображений и Ken Burns эффектов.
Содержит массив сцен. Каждая сцена — объект со следующими полями:

{
  "scene_id": "001",
  "type": "kenburns" | "animate_once" | "animate_loop",
  "image_prompt": "полный промпт для Stable Diffusion / SDXL",
  "image_style_suffix": "flat 2D cartoon, clean lines, bold colors, no gradients, yellow sweater character, white round head",
  "output_resolution": "2560x1440",
  "kenburns": {                    // только для type: kenburns
    "movement": "zoom-in" | "zoom-out" | "pan-left" | "pan-right" | "pan-up" | "pan-down" | "static",
    "start_scale": 1.0,                  // начальный масштаб (1.0 = полный кадр)
    "end_scale": 1.3,                    // конечный масштаб
    "start_x": 0.0,                    // начальная позиция X (0.0 = левый край, 1.0 = правый)
    "start_y": 0.0,                    // начальная позиция Y (0.0 = верх, 1.0 = низ)
    "end_x": 0.0,                    // конечная позиция X
    "end_y": 0.0,                    // конечная позиция Y
    "duration_sec": 7,                   // длительность сцены в секундах
    "easing": "ease-in-out" | "linear" | "ease-in" | "ease-out"
  },
  "animate": {                    // только для type: animate_once / animate_loop
    "description": "что движется и как — для Luma/AnimateDiff промпта",
    "luma_prompt": "готовый промпт для Luma Dream Machine",
    "duration_sec": 6,
    "loop": false
  }
}

━━━━
ПРАВИЛА ПАРАМЕТРОВ KENBURNS (для вариативности, не штамповки)
━━━━

Движения и типичные параметры (варьировать в указанных диапазонах):

zoom-in (приближение, нагнетание, фокус на детали):
  start_scale: 1.0, end_scale: 1.2–1.4
  start_x/y: 0.5/0.5 (центр) или смещение к объекту интереса
  easing: ease-in-out (стандарт) / ease-in (нагнетание)

zoom-out (отдаление, раскрытие, облегчение):
  start_scale: 1.3–1.5, end_scale: 1.0
  easing: ease-out (стандарт) / ease-in-out

pan-right (движение вправо, следование за персонажем):
  start_scale: 1.2–1.3, end_scale: 1.2–1.3 (масштаб не меняется)
  start_x: 0.1–0.2, end_x: 0.7–0.9
  easing: linear (стандарт) / ease-in-out

pan-left (движение влево):
  start_scale: 1.2–1.3, end_scale: 1.2–1.3
  start_x: 0.7–0.9, end_x: 0.1–0.2
  easing: linear / ease-in-out

pan-up (движение вверх, подъём):
  start_scale: 1.2–1.3, end_scale: 1.2–1.3
  start_y: 0.7–0.9, end_y: 0.1–0.2
  easing: ease-in-out

pan-down (движение вниз, тяжесть, давление):
  start_scale: 1.2–1.3, end_scale: 1.2–1.3
  start_y: 0.1–0.2, end_y: 0.7–0.9
  easing: ease-in-out

static (без движения, для акцента или анимированных сцен):
  start_scale: 1.0, end_scale: 1.0
  easing: linear

Длительность kenburns: 5–10 секунд. Варьировать: короткие (5 сек) для быстрых моментов, длинные (8–10 сек) для эмоциональных.

Комбо-движения (zoom + pan одновременно):
  Разрешены. Пример: zoom-in + pan-right = start_scale 1.0 end_scale 1.25, start_x 0.2 end_x 0.6
  В поле "movement" писать: "zoom-in+pan-right"

━━━━
ТИПЫ СЦЕН И ДЛИТЕЛЬНОСТЬ
━━━━

ТИП 1: kenburns (~80% сцен)
Статичное изображение + движение камеры через FFmpeg.
Изображение генерируется в 2560x1440 для запаса при движении.
Длительность: МАКСИМУМ 10 секунд (~25 слов TTS).
Новая сцена открывается каждые 20-25 слов TTS.

ТИП 2: animate_once (~15% сцен)
Для ключевых моментов: поворот сюжета, кульминация, юмор.
Изображение генерируется → загружается в Luma Dream Machine вручную.
В scenes.json: поле "animate" с luma_prompt.
Персонаж выполняет действие один раз: встаёт, падает, смеётся, машет рукой.
Длительность: МАКСИМУМ 15 секунд (~37 слов TTS). loop: false.

ТИП 3: animate_loop (~5% сцен)
Для растяжки сцены на 15–20+ секунд.
Фоновое движение: дыхание, пульсация, дождь, мигание.
В scenes.json: поле "animate" с loop: true.

МИНИМАЛЬНОЕ КОЛИЧЕСТВО СЦЕН:
- Видео 7-10 мин → минимум 55 сцен
- Видео 10-15 мин → минимум 75 сцен
- Скорость речи: ~150 слов/мин = ~2.5 слов/сек
- Правило: новая сцена каждые 15-25 слов TTS
- ЗАПРЕЩЕНО делать сцены длиннее 15 секунд (кроме animate_loop)

━━━━
ПРАВИЛА ПРОМПТОВ ДЛЯ ИЗОБРАЖЕНИЙ
━━━━

ОБЯЗАТЕЛЬНО добавлять в КАЖДЫЙ image_prompt:
"No empty borders or padding around the edges. Full bleed composition, image fills the entire frame edge to edge."

Пример правильного промпта:
"Cartoon character in yellow sweater sitting alone in a cinema, watching a movie, all other seats empty, a single tear runs down their cheek, expression deeply moved, screen glow lights their face softly, dark blues and warm screen light. No empty borders or padding around the edges. Full bleed composition, image fills the entire frame edge to edge."

━━━━
ПРАВИЛА ОБЛОЖКИ (thumbnail) + THUMBNAIL СЦЕНА
━━━━
Стиль: flat 2D cartoon, 16:9
Обязательно: крупный текст, высокий контраст, эмоция персонажа
Цвета: жёлтый #FFE600, белый, тёмно-синий фон
Шрифт: Impact или Anton, без градиентов
Плашка: 2-3 слова максимум, передаёт суть без названия
Персонаж: выражение лица соответствует теме (удивление, задумчивость, скептицизм)

THUMBNAIL КАК ПЕРВАЯ СЦЕНА:
Thumbnail промпт ОБЯЗАТЕЛЬНО включается как scene_id "000" в scenes.json.
В HTML файле thumbnail описывается первой сценой перед скриптом.
Тип сцены для thumbnail: kenburns, movement: static, duration_sec: 3.
Thumbnail prompt включается в HTML файл отдельным блоком метаданных.
